      • Web 前端加载这种多文件结构比较麻烦；

    JSON（JavaScript Object Notation）是一种文本数据格式。

    基本结构：{key: value}
    JSON中的value可以是String、number、object、array、boolean、null等类型

    GeoJSON可以理解为专门用来表达地理空间数据的JSON。

    • GeoJSON的最外层是一个单独的对象，这个对象可以表示
      • 几何体Geometry：只表示几何
      • 特征Feature：一个带属性的地理对象
      • 特征集合FeatureCollection：多个Feature的集合
    • 常见的几何类型
      • Point-点
      • MultiPoint-多点
      • LineString-线
      • MultiLineString-多线
      • Polygon-面
      • MultiPolygon-多面
      • GeometryCollection-几何集合

    TopoJSON是GeoJSON 按拓扑学编码后的扩展形式

    GeoJSON 直接存每个面自己的坐标，所以相邻多边形的公共边可能会重复存储。

    TopoJSON 会把共享边抽出来，变成 arcs弧，公共边只存一次。

    KML概述¶

    KML是Keyhole Markup Language，它是基于 XML 的地理数据格式，用来描述和存储地理信息数据（点、线、面、多边形、多面体以及模型等），经常用于Google Earth,Google Maps。

    KMZ 是压缩版 KML。

    CZML概述¶

    CZML是一种JSON格式的字符串，用于描述与时间有关的动画场景，CZML包含点、线、地标、模型和其他的一些图形元素，并指明了这些元素如何随时间而变化。

    是 Cesium 常用的一种动态场景数据格式

    • Cesium 中 DataSourceCollection 类是 DataSource 实例的集合。
    • 每个 viewer 自带一个 dataSources，它专门保存这个地图里已经加载的数据源。（Viewer 类中的 dataSources 属性是当前 Viewer 中 DataSource 实例的集合，是 DataSourceCollection 类的实例。）
    • DataSourceCollection 类为我们提供了一系列操作集合中数据源的方法，通过这些方法，我们可以实现对集合中 DataSource 的添加、删除、更改、查询等操作。

    dataSources 是属性名，DataSourceCollection 是这个属性对应的类型

    DataSourceCollection类常用方法

    方法 作用
    add(dataSource) 添加数据源
    remove(dataSource, destroy) 删除一个数据源
    removeAll(destroy) 删除所有数据源
    get(index) 按索引获取
    getByName(name) 按名称获取
    contains(dataSource) 判断是否包含
    raise(dataSource) 提高显示层级
    lower(dataSource) 降低显示层级

    实体数据及其可视化¶

    Entity API¶

    • Cesium 有两类用于展示空间数据的 API
      • 图元（Primitive）API：面向图形开发人员的低级API，主要目标是暴露手头执行任务所需的最小抽象量
      • 实体（Entity）API：用于数据驱动的可视化的高级API，目的是公开一组设计一致的高级对象

    实体Entity API通过Cesium的Entity实现，实际上是对Primitive API的二次封装。

    Cesium支持的实体类型：

    Entity 属性 图形类型
    point 点
    billboard 广告牌/图标
    label 标签
    polyline 折线
    polygon 多边形
    rectangle 矩形
    box 盒子
    cylinder 圆柱/圆锥
    ellipsoid 椭球
    model glTF 模型
    path 路径
    wall 墙
    corridor 走廊
    ellipse 椭圆
    plane 平面
    polylineVolume 管状线体
    tileset 3D Tiles
    • 使用DataSource向Viewer中添加矢量数据时，Cesium会将其中的数据解析为一个个实体，也可以通过Viewer类的entities属性手动加载各种实体。
    • Viewer类的entities属性是EntityCollection类的实例， EntityCollection类则是Entity的集合，EntityCollection类的add(entity)方法用向于EntityCollection中添加Entity。

    添加实体 

    1
    viewer.entities.add(...)

    删除实体 

    1
    viewer.entities.add(...)

    按id删除 

    1

    glTF模型介绍与加载¶

    glTF介绍¶

    glTF，全称是 Graphics Language Transmission Format（图形语言传输格式），是一种针对GL(WebGL，OpenGL ES以及OpenGL)接口的运行时资产（Asset）传递格式。可以理解为Web 三维模型的通用格式。

    • 目前，glTF 3D模型格式有两种：
      • *.gltf： 基于JSON的文本文件，可使用文本编辑器轻松编辑，通常会引用外部文件，例如纹理贴图、二进制网格数据等；
      • *.glb： 是二进制格式，通常文件较小且自包含所有资源，但不容易编辑。
    • glTF的核心是一个JSON文件，另外还支持外部数据。具体而言，一个glTF模型可包括以下三部分内容：
      • JSON格式的.gltf文件：其中包含完整的场景描述，并通过场景结点引用网格进行定义 。包括：节点层次结构、材质（定义了3D对象的外观）、相机（定义义了渲染程序的视锥体设置 ）、mesh（网格）、动画（定义了3D对象的变换操作，比如选择、平移操作）、蒙皮（定义了3D对象如何进行骨骼变换）等。
      • .bin文件：包含几何和动画数据以及其他基于缓冲区的数据的二进制文件。
      • 图像文件（.jpg，.png）。

    3DTiles介绍与加载¶

    3D Tiles 是 Cesium 推出的开放规范，用于流式传输大规模异构 3D 地理空间数据集

    • Cesium中定义了许多3D Tiles相关的类，其中有几个比较常用的类：
      • Cesium3Dtileset：用于流媒体大量的异构3D地理空间数据集
      • Cesium3DTileStyle：瓦片样式
      • Cesium3DTile:数据集中的一个瓦片
      • Cesium3DTileContent：瓦片内容
      • Cesium3DTileFeature：数据集要素（提供对存储在tile的batch table中的特性属性的访问）
    • Cesium也可以通过Entity API 或 Primitive API加载3D Tiles数据，但因为多数情况下3D Tiles数据都是成片区的数据，数据量比较大，所以为了保证性能，建议使用Primitive API。

    Primitive API¶

    Primitive API 是更底层的图形 API，更接近渲染引擎。它要求开发者更像图形程序员一样思考

    • Primitive两个核心组成
      • GeometryInstance：定义几何结构
      • Appearance：定义外观、材质、着色方式
    • 优势：
      • 性能：绘制大量Primitive时，可以将其合并为单个Geometry以减轻CPU负担并更好地使用GPU。合并Primitive由web worker线程执行，以保持UI响应性；
      • 灵活性：Geometry与Appearance 解耦，两者可以分别进行修改；
      • 低级别访问：易于编写GLSL顶点、片段着色器、使用自定义的渲染状态 。
    • 劣势：
      • 需要编写更多的代码，并且需要开发者对图形编程有更深刻的理解，尤其是OpenGL知识；
      • 需要对组合几何形状对于静态数据有效，而对于动态数据则不一定有效。

    为什么性能高：
    Primitive 绘制大量对象时，可以把多个几何体合并，以减少 CPU 负担并更好利用 GPU。
    一个Primitive可以有多个几何实例，但只能有一个外观。几何图形定义了结构，外观定义了每个像素被如何着色，外观可能使用材质（Material）
    例如画 3000 个矩形，如果用 Entity，Cesium 要管理很多高级对象，如果用 Primitive，可以把很多 GeometryInstance 合成一个 Primitive。

    • 外观：
      • flat：扁平化着色，不考虑光线的作用
      • faceForward：布尔值，控制光照效果

    粒子系统¶

    • 粒子系统中的粒子通过粒子发射器生成。
      • 粒子发射器产生的粒子都有一个位置和类型，存活一段时间后然后消亡。
      • 粒子发射器则控制了粒子产生时的初始位置、速度和方向，并依据粒子发射频率（emissionRate）来决定每秒产生多少粒子，通过发射器类型决定粒子的发射速度和方向。
    • Cesium内置了四种粒子发射器，分别为：BoxEmitter、CircleEmitter、ConeEmitter、SphereEmitter。这些发射器类都继承了基类ParticleEmitter
      • BoxEmitter：盒形发射器。定义一个盒型，将粒子随机地放置在盒子里的随机一个位置，并且具有从盒子中心发出的初始速度，然后沿着盒子的6个面的法向量向外运动。
      • CircleEmitter：圆形发射器。定义了一个圆形，在圆形面中的随意一个位置发射具有沿着z矢量的初始速度的粒子，是Cesium默认的发射器。
      • ConeEmitter：锥形发射器。定义一个锥体，在锥体顶点产生粒子，并向锥体内随机一个方向运动。
      • SphereEmitter：球形发射器。定义一个球体，在球体内随机产生粒子，并沿着球心向外运动。

    Property¶

    Property 是 Cesium 用来描述“属性值如何随时间变化”的机制。 它主要服务于 Entity API。
    Primitive 体系一般不直接使用 Property，而是更偏向 Geometry、Appearance、Material 这些底层对象。

    Property 是 Cesium 中用于描述属性值的对象，它可以返回某一时间点的属性值，因此可以支持动态可视化

    Property 本身是抽象类，不能直接写成new Cesium.Property()，而应该用它的子类，如new Cesium.ConstantProperty(...)

    核心类

    名称 含义
    isConstant 判断这个属性是否是常量
    definitionChanged 当属性定义发生变化时触发事件
    getValue(time) 获取某个时间点的值
    equals(other) 判断两个 Property 是否相等

    基本Property
    基本 Property 主要控制普通属性值，比如数字、颜色、布尔值等。

    ConstantProperty:值不会随时间的变化而变化，Cesium内部会隐晦地将你所定义地属性（包括Property、MaterialProperty、PositionProperty等）转成ConstantProperty。
    color: Cesium.Color.RED和color: new Cesium.ConstantProperty(Cesium.Color.RED)在效果上可以理解为类似

    SampledProperty：通过给定多个不同时间点的Sample，然后在每两个时间点之间进行插值的一种Property。

    TimeIntervalCollectionProperty：和 类似，不过SampledProperty实现的效果是平滑的变化；而TimeIntervalCollectionProperty按照每个时间段来修改Property，单独每一个时间段内属性值是不变的，实现效果是跳跃式的变化。

    CompositeProperty：组合的Property，在单个类型的Property无法满足实际的需求时，可以把ConstantProperty、SampleProperty、TimeIntervalCollectionProperty等多种类型的Property组合在一起进行实现。比如前一个时间段需要线性运动，后一段时间再跳跃式运动。

    MeterialProperty
    MaterialProperty是用来专门表示材质的Property，它对Property进行了扩展，增加了getType方法，用来获取材质类型。 MaterialProperty和Property一样，是一个抽象类，不能直接被实例化。

    PositionProperty

    • PositionProperty是专门用来表示Position的Property，扩展了Property的接口。和Property一样，是一个抽象类，不能直接被实例化。和普通的Property本质上是一致的，用法上也大同小异，只是多了一个Position。
    • 以SampledPositionProperty为例，实现动态更改位置的效果

    其他Property
    - CallbackProperty：自由度最高的一种Property，可以让用户通过自定义回调函数来返回需要的值 - ReferenceProperty：可以直接链接到别的对象的Property上，相当于引用，不用自己再构建一次 - PropertyBag：可以包装一个对象，该对象的每一个属性都可以作为一个动态的Property - PropertyArray：和PropertyBag类似，只是其内部封装了一个数组 - VelocityOrientationProperty：用Entity的position的位置变化，来计算出移动的方向，最后把速度方向输出成Orientation

    概念 主要服务对象 作用
    Entity 高级 API 表示空间实体
    Primitive 底层 API 高性能绘制几何
    Property Entity 表示属性值，支持随时间变化
    MaterialProperty Entity 表示 Entity 的动态材质
    PositionProperty Entity 表示 Entity 的动态位置
    Material Primitive 表示 Primitive 的材质
    Appearance Primitive 表示 Primitive 的外观和渲染方式
