    什么是机器学习¶

    机器学习是人工智能和计算机科学的一个分支，它关注如何使用数据和算法来模仿人类学习的方式，并逐渐提高模型的准确性。

    人工智能 AI ⊃ 机器学习 ML ⊃ 深度学习 DL

    人工智能 Artificial Intelligence是最外层概念，指能够感知、推理、行动和适应的程序。人工智能的概念最早可以追溯到 1956 年左右。
    机器学习 Machine Learning是人工智能的一个子领域。它的特点是算法性能会随着接触更多数据而提升。机器学习大约从 1980s 开始发展起来。
    深度学习 Deep Learning是机器学习的一个子领域，主要利用多层神经网络从大量数据中学习特征，尤其在 2010s 之后快速发展。

    机器学习如何工作¶

    • 决策过程：一般而言，机器学习算法用于进行预测，分类或模式识别。
    • 误差函数：误差函数用于评估模型的预测。如果存在已知示例，误差函数可以进行比较以评估准确性。
    • 模型优化过程：如果模型能够更好地拟合训练集中的数据点，则调整权重以减少已知示例与模型估计之间的差异。算法将重复此评估和优化过程，自主更新权重，直到达到准确性阈值。
      • 预测 → 计算误差 → 调整参数 → 再预测 → 再调整

    机器学习的三大类型¶

    监督机器学习¶

    • 监督机器学习（或监督学习）的定义是使用标记数据集来训练算法，以对数据进行分类或预测结果。
    • 当输入数据被送入模型时，模型调整其权重直到拟合完成。这作为交叉验证过程的一部分，以确保模型避免过拟合或欠拟合。
    • 监督学习中使用的一些方法包括：神经网络、朴素贝叶斯、线性回归、逻辑回归、随机森林、支持向量机（SVM）等。

    无监督机器学习¶

    • 无监督机器学习（无监督学习），使用机器学习算法来分析和聚类未标记的数据。
    • 这类算法发现隐藏的模式或数据分组，无需人工干预。其发现信息中相似性和差异性的能力使其成为探索性数据分析、交叉销售策略、客户细分、图像和模式识别的理想解决方案。
    • 它还可用于通过降维过程减少模型中的特征数量；主成分分析（PCA）和奇异值分解（SVD）是两种常见方法。
    • 无监督学习中使用的其他算法包括神经网络、K均值聚类、ISODATA、概率聚类方法等。

    半监督学习¶

    • 半监督或少样本学习提供了监督学习和无监督学习之间的折中方案。
    • 在训练过程中，它使用较小的标记数据集来指导从较大的未标记数据集中进行分类和特征提取。
    • 半监督学习可以解决标记数据不足（或无法承担标记足够数据的成本）来训练监督学习模型的问题。

    随机森林分类¶

    决策树¶

    决策树是一种通过一系列“如果……那么……”的判断规则，把样本逐步划分到不同类别或预测数值的模型。

    概念¶

    决策树是一种决策支持工具，它用树状结构表示决策过程及其可能后果，包括事件结果、成本和效用等。

    在机器学习分类中，决策树可以理解为一套不断判断的规则：如果条件 A 成立，就往左走；否则往右走。再判断条件 B，继续分支。最后到达叶节点，得到分类结果

    决策树中有两种类型的节点：

    • 内部节点：根据相应属性可以取的不同值将数据分成不同的分支。
    • 叶节点：代表一个类别或决策结果。

    构建决策树的机制-自顶向下的贪婪搜索方法

    • 选择最佳属性 \(a^*\) 放置在树的根节点
    • 将训练集 \(D\) 分成子集 \(\{D_1, D_2, \ldots, D_k\}\)，其中每个子集 \(D_i\) 包含具有相同 \(a^*\) 值的示例。
    • 递归地对每个新子集应用该算法，直到示例具有相同的类别或数量很少。

    数据分割¶

    什么属性/特征是分割数据的最佳选择？

    如果某个属性/特征分类结果的复杂度降低得更多，则该特征更优。

    ID3：使用信息熵和信息增益（IG）分割数据¶

    ID3是典型的决策树算法，也是C4.5和C5.0算法的基础，它使用信息熵和信息增益（IG）来测量哪个属性/特征是分割数据的最佳选择。

    信息熵：

    • 与随机样本集 \(S\) 相关的不确定性或熵度量定义为：
    \[H(S) = -\sum_{i=1}^{K} p_i \log_2 p_i\]
    • 其中 \(p_i\) 指随机过程中第 \(i\) 个可能结果或事件的概率。
    • 熵越大，数据或分类结果越复杂。

    信息增益：

    信息增益（IG）表示两个信息熵之间的差异。它用于在决策树算法中选择特征。

    • 示例集 \(S\) 在属性 \(X\) 上的信息增益：
    \[IG(S, X) = H(S) - \sum_{v \in Values(X)} \frac{|S_v|}{|S|} H(S_v)\]
    • \(H(S)\)：所有示例 \(S\) 的熵
    • \(Values(X)\)：属性 \(X\) 的所有可能值
    • \(H(S_v)\) 是子样本 \(S_v\) 的熵
    • "最佳"测试值就是使 \(IG(S, X)\) 最大化的选择。信息增益（IG）越大，特征的选择性越好。
    example

    假设有 10 个样本，目标变量是“是否打球”：

    天气 打球
    晴天 不打
    晴天 不打
    晴天 打
    阴天 打
    阴天 打
    阴天 打
    雨天 打
    雨天 不打
    雨天 不打
    雨天 打

    总体中：

    \(打=6,\quad 不打=4\)

    所以总体熵为：

    \[ H(S)=-\left(\frac{6}{10}\log_2\frac{6}{10}+\frac{4}{10}\log_2\frac{4}{10}\right) \]
    \[ H(S)=0.971 \]

    按“天气”划分：

    晴天有 3 个样本：

    \(打=1,\quad 不打=2\)

    \[ H(S_{\text{晴天}})=- \left( \frac{1}{3}\log_2\frac{1}{3} + \frac{2}{3}\log_2\frac{2}{3} \right) =0.918 \]

    阴天有 3 个样本：

    \(打=3,\quad 不打=0\)

    \[ H(S_{\text{阴天}})=0 \]

    雨天有 4 个样本：

    \(打=2,\quad 不打=2\)

    \[ H(S_{\text{雨天}})=- \left( \frac{2}{4}\log_2\frac{2}{4} + \frac{2}{4}\log_2\frac{2}{4} \right) =1 \]

    计算按“天气”划分后的加权熵：

    \[ H(S|\text{天气})= \frac{3}{10}(0.918) + \frac{3}{10}(0) + \frac{4}{10}(1) \]
    \[ H(S|\text{天气})=0.675 \]

    计算信息增益：

    \[ IG(S,\text{天气})=H(S)-H(S|\text{天气}) \]
    \[ IG(S,\text{天气})=0.971-0.675=0.296 \]

    结论：用“天气”划分后，熵从 \(0.971\) 降低到 \(0.675\)，信息增益为 \(0.296\)，说明“天气”这个特征可以减少分类不确定性。

    过拟合Overfitting¶

    过拟合：模型在训练数据上表现很好，但学到了过多训练样本中的噪声、偶然性或细节特征，导致它在新的、未见过的数据上表现变差的现象。

    • 决策树应该长多深？
      • 如果树长得太深，它可能不仅学习了数据中的真实规律，还学习了噪声和偶然偏差。
      • 例如图中某些异常点可能只是噪声，但如果树继续分裂，就会专门为这些噪声点建立规则。这会导致过拟合。

    过拟合：当树t在数据集D上的分类准确率比另一个树t'更好，但在D'上的分类准确率比t'更差时，树t过拟合了数据集 D。

    • 数据过拟合的原因：
      • 随机误差或噪声：示例具有错误的类别标签或错误的属性值。
      • 偶然模式：由于数据集规模较小，示例偶然偏离某种模式。
    • 解决过拟合
      • 提前停止树：在树开始过拟合数据之前提前停止。在实践中，这种解决方案难以实施，因为不清楚什么是好的停止标准。
      • 生长树直到算法停止：即使出现过拟合问题，也让树生长。然后作为后处理步骤对树进行剪枝。这种方法已被证明非常有效。

    剪枝的目的不是让训练精度最高，而是提升泛化能力，也就是对新数据的预测能力。

    决策树的典型算法¶

    • ID3（二分法），1986年，Ross Quinlan

      • 通过信息增益选择最优特征进行节点分裂，递归构建决策树，只能用于离散特征处理，无缺失值处理，无剪枝。

    • C4.5，1993年，Ross Quinlan

      • ID3的改进版，引入增益率，支持连续特征和缺失值处理，加入剪枝。

    • C5.0，1998年，Ross Quinlan

      • C4.5的改进版，改进了连续特征、缺失值和剪枝处理。

    • CART（分类与回归树），1984年，Leo Breiman

      • 使用基尼指数（Gini Index）代替熵，支持回归任务。
      • \(Gini(S) = 1 - \sum_{i=1}^{K} p_i^2\)
      • S：数据集；K：类别总数；\(p_i\)：第i类样本在数据集S中的占比。
      • 基尼指数用于反映数据集中类别的分布均匀程度：
        • 值越小，数据纯度越高（某个类别占主导）
        • 值越大，数据越不纯（类别分布越均匀）

    集成方法¶

    集成方法是指把多个基础模型组合起来，共同完成分类或回归任务，以获得比单个模型更稳定、更准确的预测结果。

    • 集成方法
      • PAC（概率近似正确），1984年，理论基础
      • 弱学习和强学习，1994年
      • Boosting，1996年，串行模式
      • Bagging，1997年，并行模式
      • 随机森林，2001年
    • 集成学习中的两个问题：
      • 如何训练基分类器？
      • 如何融合基分类器？

    Bagging¶

    Bagging 就是：随机抽很多份训练数据，分别训练很多个模型，最后投票或平均。

    Bagging（Bootstrap聚合）是一种降低方差的方法。Bootstrap是一种有放回的抽样方法

    Bagging通过从原始数据集中进行有放回的随机抽样来生成多个训练子集，每个子集都用于训练一个基分类器/基回归器，再通过集成这些基分类器/基回归器预测来提高整体预测的性能和稳定性。

    Bagging过程：用不同的随机样本训练多个模型，再让它们投票或平均，降低不稳定性。

    • 生成Bootstrap样本
      • 从数据集 \(D\) 中有放回地进行 \(B\) 次样本抽取，生成 \(B\) 个Bootstrap样本 \(\{D_1, D_2, \ldots, D_B\}\)，\(B\) 通常取100-500。未被抽中的样本为OOB（Out-of-Bag，袋外样本）。
    • 构建和训练基学习器
      • 使用每个Bootstrap样本 \(D_b\) 构建和训练一个基学习器（决策树、神经网络等）。
    • 聚合所有基学习器的结果
      • 分类：多数投票
      • 回归：取平均值

    随机森林是 Bagging 的一种典型方法。

    Boosting¶

    Boosting 是一个模型接一个模型地训练，后一个模型重点纠正前一个模型犯的错误。

    Boosting是一种通用方法，可应用于许多统计学习方法进行回归或分类。

    • 每次使用的是全部的样本，每轮训练改变样本的权重。
    • 下一轮训练的目标是找到一个函数f来拟合上一轮的残差。
    • 当残差足够小或者达到设置的最大迭代次数则停止。

    顺序训练：基学习器按顺序生成，后续模型关注前序模型的错误。

    权重调整：根据前序模型的预测误差，动态调整样本权重或损失函数，使后续模型更关注难样本。

    聚合方式：通过加权投票或加权求和集成结果。

    XGBoost、AdaBoost、GBDT（梯度提升决策树）和LightGBM都是Boosting算法。

    随机森林¶

    随机森林由很多棵决策树组成，每棵树用随机抽样的数据和随机选择的特征来训练，最后通过投票或平均得到最终结果。

    Note

    名词解释：随机森林分类 简答题：简述用随机森林分类器识别建设用地的流程

    随机森林是一种集成学习方法，它基于多棵分类与回归树 CART 的组合。

    • 每棵树通过独立地抽样训练数据集和一组变量来进行训练，通过Bagging聚合。
    • 随机森林通过使用随机输入特征子集的最佳分割而不是使用最佳分割变量来生长树，这可以减少树之间的相关性和泛化误差。基尼指数通常用于随机森林中作为最佳分割选择的度量，它最大化不相似性。
    • 随机森林具有处理大数据集和变量的许多优点，但不会过拟合。

    • 随机森林分类器中有两个参数：每个节点随机子集中的活跃变量数（\(M\)）和森林中的树数（\(T\)）。

    • 随机森林有两个层面的随机性

      • 样本随机：每棵树使用 Bootstrap 抽样得到不同训练子集。
      • 特征随机：每个节点分裂时，不是从全部 M 个特征中找最优特征，而是随机抽取 m 个特征，再在这 m 个特征中找最优划分。
        • 降低树与树之间的相关性，从而降低泛化误差。
        • 随机森林通常使用 Gini 指数（基尼指数）来选择最优分裂。

    • 随机森林的重要参数

      • M 或 p：总输入变量数
      • m：每个节点随机选择的特征数
        • 每次节点分裂只从 m 个随机特征中选择最佳分裂，一般 m 小于 M
      • T 或 B：森林中的树数量
        • 树越多，模型通常越稳定，但计算量越大。

    • 随机森林生长过程

      1. 如果训练集有 N 个样本，就从原始数据中有放回随机抽取 N 个样本；
      2. 如果有 M 个输入变量，则每个节点随机选择 m 个变量；
      3. 在这 m 个变量中选择最优分裂；
      4. 每棵树尽可能生长到最大；
      5. 不剪枝。

    算法流程¶

    对于 b = 1 到 B 棵树：

    1. 从训练数据中抽取一个大小为 N 的 Bootstrap 样本 Db；
    2. 在 Db 上生长一棵随机森林树；
    3. 对每个终端节点递归执行：
      • 从 M 个变量中随机选择 m 个；
      • 在 m 个变量中选择最佳变量和最佳分裂点；
      • 把节点分成两个子节点；

    4. 输出所有树组成的集成模型。 预测新样本 x 时：

    5. 回归任务：对所有树结果求平均；

    6. 分类任务：对所有树结果多数投票。

    误差¶

    误差原因：

    • 树之间的相关性：树与树越相似，相关性越高，森林误差率越高。
    • 单科树的强度：单棵树越强，错误率越低，森林整体误差率越低。

    矛盾：
    减小 m 会降低树之间相关性，但也可能降低单棵树强度；增大 m 会增强单棵树，但也会提高树之间相关性。因此 m 需要调参。

    调参¶

    • 建议
      • 对于分类，\(m\) 的默认值是 \(\sqrt{p}\)，最小节点大小为1。
      • 对于回归，\(m\) 的默认值是 \(p/3\)，最小节点大小为5。
    • 在实践中，这些参数的最佳值将取决于问题，它们应被视为调参参数。
    • 与Bagging一样，我们可以使用OOB，因此随机森林可以在一个序列中拟合，同时进行交叉验证。一旦OOB误差稳定，训练就可以停止。

    优点与局限¶

    优点：

    • 精度高
    • 适合大数据
    • 能处理大量变量
    • 能评估变量重要性
    • 在森林构建过程中生成泛化误差的无偏内部估计。
    • 能处理缺失数据，并在大量数据缺失时保持准确性。
    • 适合类别不平衡问题
    • 可保存模型以供将来在其他数据上使用。
    • 可以计算样本相似性
      • 计算案例对之间的邻近度，可用于聚类、定位异常值，或通过缩放提供有趣的数据视图。
    • 可扩展到无标签数据

    局限性：

    • 噪声数据上仍可能过拟合
    • 类别变量水平数不同会产生偏差
      • 对于包含不同级别数量的分类变量的数据，随机森林偏向于那些级别更多的属性。因此，对于这类数据，随机森林的变量重要性分数不可靠。

    OOB袋外误差估计¶

    Bootstrap 抽样是有放回抽样，因此每棵树训练时并不会用到所有样本。平均大约有三分之一的样本不会进入某棵树的训练集，这些样本就是OOB（Out-of-Bag samples）

    OOB 误差就是用“没参与训练的样本”来估计随机森林的错误率。
    在随机森林中，无需交叉验证或单独的测试集来获得测试集误差的无偏估计。它在运行过程中内部估计，方法如下：

    • 每棵树使用来自原始数据的不同Bootstrap样本构建。约三分之一的案例被排除在Bootstrap样本之外，不用于构建第 \(k\) 棵树。
    • 将每个在构建第 \(k\) 棵树时被排除的案例放入第 \(k\) 棵树中进行分类。
    • 以这种方式，每个案例在约三分之一的树中获得测试集分类。
    • 在运行结束时，取 \(j\) 为每次案例 \(n\) 为OOB时获得最多票数的类别。
    • \(j\) 不等于案例 \(n\) 真实类别的次数比例，对所有案例取平均，就是OOB误差估计。
    • 这在许多测试中被证明是无偏的。

    变量重要性、交互¶

    如果破坏某个变量后模型表现变差很多，这个变量就很重要。
    如果变量数量特别多，可以先用全部变量运行一次随机森林，找出重要变量，再只用重要变量重新运行模型。

    如果一个变量被用来分裂之后，会影响另一个变量是否也被用来分裂，那么这两个变量之间可能有交互关系。

    tree 的 split 就是决策树根据某个变量和阈值，把数据分成两组。

    评估指标¶

    • 精确率（Precision） ：\(Precision = \frac{TP}{TP + FP}\)
      • 正确正例（TP）占总预测正例（TP + FP）的比例，
    • 召回率（Recall） ：\(Recall = \frac{TP}{TP + FN}\)
      • 正确正例（TP）占总实际正例（TP + FN）的比例。
    • Kappa： \(Kappa = \frac{P_o - P_e}{1 - P_e}\)
      • 其中 \(P_o\) 是观察到的一致性，\(P_e\) 是期望的一致性。
      • Kappa 越高，分类一致性越好，Kappa 比总体精度更严格一些。
    • IoU： IoU = TP / (TP + FP + FN)
      • IoU 常用于分割或空间分类，表示预测区域和真实区域的重叠程度。

    TP (True Positive): 真的正样本，即正样本被正确分类为正样本
    TN (True Negative): 真的负样本，即负样本被正确分类为负样本
    FP (False Positive): 假的正样本，即负样本被错误分类为正样本
    FN (False Negative): 假的负样本，即正样本被错误分类为负样本
