    空间聚类分析¶

    约 3336 个字 预计阅读时间 13 分钟

    什么是聚类分析¶

    聚类 cluster 是一组数据对象、地理要素或点。一个好的簇内部对象应该彼此相似，而不同簇之间应该明显不同。

    聚类分析 clustering analysis 是根据数据本身的特征发现相似性，并把相似的数据对象归为一类。
    通常是无监督学习。

    • 聚类分析的两类典型应用
      • 作为独立工具来获取数据分布的洞察
      • 作为其他算法的预处理步骤

    如何度量聚类的相似性-基于距离的方法和基于距离的方法¶

    一个好的聚类方法应该满足两个条件：类内相似性高，类间相似性低。

    基于距离的方法¶

    闵可夫斯基距离

    \[d(i,j) = (|x_{i1}-x_{j1}|^q + |x_{i2}-x_{j2}|^q + ... + |x_{ip}-x_{jp}|^q)^{1/q}\]

    其中 \(x_{i1}, x_{i2}, ..., x_{ip}\) 和 \(x_{j1}, x_{j2}, ..., x_{jp}\) 是两个p维数据对象，q是正整数

    q = 1，曼哈顿距离（城市街区距离）

    \[d(i,j) = |x_{i1}-x_{j1}| + |x_{i2}-x_{j2}| + \cdots + |x_{ip}-x_{jp}|\]

    适合描述网格路网中的距离，比如城市街区中不能走直线，只能沿东西南北方向走。

    q=2，欧氏距离

    \[d(i,j) = \sqrt{(x_{i1}-x_{j1})^2 + (x_{i2}-x_{j2})^2 + \cdots + (x_{ip}-x_{jp})^2}\]

    欧式距离性质：

    • \(d(i,j) \geq 0\)
    • \(d(i,j) = 0\) 当且仅当 \(i=j\)
    • \(d(i,i) = 0\)
    • \(d(i,j) = d(j,i)\) （对称性）
    • \(d(i,j) \leq d(i,k) + d(k,j)\) （三角不等式）

    计算聚类间距离的典型替代方法：

    • 最小距离：两个簇中任意两个对象之间的最小距离，也叫 single linkage。它容易把链状对象连接起来。
    • 最大距离：两个簇中任意两个对象之间的最大距离，也叫 complete linkage。它更强调整个簇之间都要接近。
    • 平均距离：两个簇所有对象两两距离的平均值。
    • 质心距离：Centroid distance两个簇中心点之间的距离。
    • 中心对象距离 medoid distance：两个簇中代表性中心对象之间的距离。medoid 必须是簇内真实存在的对象，而 centroid 可以是均值点，不一定对应真实对象。

    基于相似系数的方法¶

    夹角余弦和相关系数 \(r\) 是两类常用的相似系数。

    夹角余弦

    夹角余弦用于衡量两个向量方向是否接近：

    \[\cos \theta_{ij} = \frac{\sum_{k=1}^{n} x_{ik} \cdot x_{jk}}{\sqrt{\sum_{k=1}^{n} x_{ik}^2} \cdot \sqrt{\sum_{k=1}^{n} x_{jk}^2}}\]

    其中 \(i\) 和 \(j\) 是两个对象的序号，\(x_k\) 代表第 \(k\) 个变量。

    相似系数矩阵

    \(\(\theta = \begin{pmatrix} \cos\theta_{11} & \cos\theta_{12} & \cdots & \cos\theta_{1m} \\ \cos\theta_{21} & \cos\theta_{22} & \cdots & \cos\theta_{2m} \\ \cdots & \cdots & \cdots & \cdots \\ \cos\theta_{m1} & \cos\theta_{m2} & \cdots & \cos\theta_{mm} \end{pmatrix}\)\)

    • \(\theta_{ij} \in [-1, 1]\)
    • 夹角余弦值越大，对象越相似。

    ==空间聚类==方法¶

    Note

    简答题：写出两种聚类分析，并说明其不同点，层次聚类思想

    空间聚类方法总览¶

    • 划分式聚类 partitioning clustering
      • 给定簇数 (k)，把数据划分为 (k) 个簇，并优化某个目标函数
      • 典型方法包括 k-means、k-medoids、ISODATA、CLARANS
    • 层次聚类 hierarchical clustering
      • 把数据构造成层次树，可以从下往上合并，也可以从上往下拆分
      • 典型方法包括 AGNES、DIANA、BIRCH、ROCK、CAMELEON
    • 密度聚类 density-based clustering
      • 根据点的密度发现簇，可以识别噪声
      • 典型方法包括 DBSCAN、OPTICS、HDBSCAN
    • 约束聚类 constraint-based clustering
      • 聚类时加入用户指定或应用相关的限制，比如空间邻接、障碍物、最小区域规模等
      • 典型方法包括空间约束聚类、COD 等
    • 模型聚类 model-based clustering
      • 假设每个簇符合某种模型，然后寻找最优拟合
      • 典型方法包括 SOM、EM、COBWEB。
    • 栅格或矢量方法
      • 空间对象可以是 raster，也可以是 vector。不同数据结构会影响聚类方法选择。

    划分聚类¶

    划分聚类：把 n 个样本直接划分成 k 个互不重叠的类别，使得同一类内部的样本尽量相似，不同类之间的样本尽量不同。

    划分聚类的目标：把包含n个对象的数据集D划分成k个簇，使簇内平方误差和最小

    \[\text{最小化} \sum_{m=1}^{k} \sum_{i=1}^{n} (C_m - v_{mi})^2\]

    \(C_m\) 是聚类 \(m\) 的中心，\(V_{mi}\) 是聚类 \(m\) 中第 \(i\) 个对象的值。

    k-means 方法¶

    • 步骤
      • 预先指定簇数k；
      • 随机选择k个对象或点作为初始中心；
      • 把每个对象分配给距离最近的中心；
      • 重新计算每个簇的均值中心；
      • 重复分配和更新，直到对象不再改变所属簇。
    • 优点：
      • 相对高效：\(O(tkn)\)，其中 \(n\) 是对象数，\(k\) 是聚类数，\(t\) 是迭代次数。通常，\(k, t \ll n\)。
    • 缺点
      • 通常终止于局部最优。
      • 无法处理噪声数据和异常值（对离群值敏感，因为极端值会显著拉动均值中心。）
      • 只能用于均值有意义的数据，分类变量不适合直接使用；
      • 需要预先指定 \(k\)，即聚类的数量
      • 不适合发现非凸形状的聚类

    k-medoids 方法¶

    medoids-中心对象

    K-Medoids：不使用聚类中对象的均值作为参考点，而是使用中心点（medoid），它是聚类中最位于中心位置的对象

    因此 k-medoids 通常比 k-means 更稳健，但计算成本也更高。

    ISOData 迭代自组织数据分析技术算法¶

    ISODATA 基于 k-means，但增加了两个操作：合并 merging 和 分裂 splitting。

    • 合并操作
      • 当两个簇中心之间距离小于阈值时，把两个簇合并；当某个簇样本数量不合适时，也可能和邻近簇合并。
    • 分裂操作
      • 当某个簇的标准差超过阈值，或者样本数量超过阈值时，把该簇拆成两个。

    ISODATA 比 k-means 更灵活，不完全固定簇数，而是可以根据数据结构动态调整类别。

    层次聚类¶

    层次聚类：按照样本之间的相似程度，逐步合并或逐步拆分样本，形成一个从细到粗或从粗到细的树状聚类结构。

    Note

    名词解释：层次聚类法 计算题：给一个矩阵，用层次聚类方法计算聚类过程

    层次聚类使用距离矩阵作为聚类依据。它不一定要求提前指定簇数 (k)，但需要一个终止条件，比如达到某个距离阈值，或者在树状图上切成指定数量的簇。

    • AGNES，Agglomerative Nesting，凝聚型层次聚类。
      • 自底向上。开始时每个对象都是一个簇，然后逐步合并最相似的簇，直到所有对象合成一个大簇。
    • DIANA，Divisive Analysis，分裂型层次聚类。
      • 自顶向下。开始时所有对象在一个簇中，然后逐步拆分，直到每个对象单独成簇或达到终止条件。

    AGNES方法¶

    在统计分析包中实现，例如SPSS、Splus

    • 步骤
      • 把每个对象看作一个单独簇；
      • 根据不相似矩阵，找出距离最小的两个簇；
      • 合并它们；
      • 重复上述过程；
      • 最终所有对象属于同一个簇。

    层次聚类的结果可以表示成一棵树，纵轴通常是距离或不相似度。想得到几个簇，就在树状图上某个高度“横切”。横切线以下的连通分支就是聚类结果。

    改进的AGNES（聚合嵌套）¶

    基本思想仍然是从距离矩阵出发，但每次合并两个簇后，要重新计算新簇与其他簇之间的距离

    1. 将 \(m \times m\) 相异性矩阵中的每个节点视为一个聚类；
    2. 使用最小距离方法和相异性矩阵将相异性最小的聚类/节点合并为一个新聚类；\(G_k=\{G_p, G_q\}\)，\(d_{pq} = \min\{d_{ij}\}\)
    3. 分别计算新聚类与其他节点/聚类之间的相异性距离，得到一个 \((m-1) \times (m-1)\) 矩阵； \(d_{rk} = \min\{d_{rp}, d_{rq}\}\) 或 \(d_{rk} = \max\{d_{rp}, d_{rq}\}\)
    4. 以非递减方式继续；
    5. 最终所有节点属于同一个聚类。

    DIANA 方法¶

    DIANA 方法 与 AGNES 方向相反，是自顶向下分裂。 AGNES 是：单个对象 → 小簇 → 大簇 → 总簇。
    DIANA 是：总簇 → 大簇 → 小簇 → 单个对象。

    AGNES 和 DIANA 不仅可以对对象聚类，也可以对变量聚类。例如可以把多个社会经济指标按相似性分组。

    基于密度的聚类¶

    基于密度聚类的核心思想¶

    • 基于密度的聚类工具通过检测点集中的区域以及被空旷或稀疏区域分隔的区域来工作。不属于聚类的点被标记为噪声。
    • 该工具使用无监督机器学习聚类算法，这些算法仅基于空间位置和到指定数量邻近点的距离自动检测模式。

    潜在应用¶

    • 城市供水管网。管道破裂点如果形成聚集，说明某些区域可能存在系统性风险，工程师可以提前维护。
    • 自然灾害或恐怖袭击后，带地理位置的推文可以被聚类，用于判断救援和疏散需求集中在哪里。
    • 虫媒疾病或虫害分析。若家庭点数据中有受害家庭和未受害家庭，可以通过密度聚类找出受害家庭最大聚集区，从而优先治理。

    三种方法¶

    • 定义距离（DBSCAN），基于固定距离的密度聚类。
      • 用指定距离把密集簇和稀疏噪声分开。
      • 优点：速度快；
      • 缺点：需要有一个清晰、统一、适合所有簇的距离阈值，
      • 更适合密度比较一致的簇。
      • DBSCAN 也可以加入时间字段和搜索时间间隔，用于时空点簇发现。
    • 自适应（HDBSCAN），层次密度聚类。
      • 使用可变距离区分不同密度的簇和噪声。
      • 数据驱动，用户输入较少，适合不同区域密度差异明显的数据。
    • 多尺度（OPTICS）
      • 使用邻近点之间的距离和可达性图将不同密度的聚类与噪声分离
      • 调参灵活
      • 计算量较大，尤其当搜索距离很大时。

    DBSCAN¶

    • 概念
      • \(\varepsilon\)-邻域：空间中任意点的 \(\varepsilon\)-邻域是以该点为圆心、\(\varepsilon\) 为半径的圆所包含的区域。
      • 点密度：点密度是 \(\varepsilon\)-邻域中包含的点数与该邻域面积的比值。\(\varepsilon\)-邻域中包含的点数可视为点数量。
      • 核心点：如果某点 \(\varepsilon\) 范围内的点数大于阈值，则该点称为核心点。
      • K阶邻域距离：一个点与其第 \(k\) 个邻近点之间的距离
      • K-Dis图：对每个点的 \(k\) 阶邻域距离进行排序的示意图。

    步骤：

    • 1.输入邻域数阈值 \(\text{MinPts}\)
    • 2.绘制K-Dis图确定 \(\varepsilon\)
    • 3.找到任意一点
    • 4.邻域内是否有核心点？
      • 否 → 定为噪声点
      • 是 → 所有核心点入栈作为种子点
    • 5.对种子点扩张：确定邻域点，将邻域内的点标记为同一类
    • 6.栈是否为空？
      • 否 → 种子点出栈，回到步骤5
      • 是 → 检查所有的点是否都被标记
        • 否 → 回到步骤3
        • 是 → 绘制聚类图，程序结束

    参数解释：

    • Minimum Features per Cluster，每个聚类的最小要素数
      • 该参数确定将一组点视为聚类所需的最小要素数。
      • 如果设得太小，原本有意义的大簇可能被切成很多小簇。
      • 该参数越小，检测到的簇越多；参数越大，检测到的簇越少。
    • core-distance 核心距离
      • 指从某个点出发，为了达到最小要素数所需要覆盖的距离
      • 如果 Minimum Features per Cluster 设得大，核心距离通常也变大；如果设得小，核心距离也变小。
      • 核心距离与 DBSCAN 和 OPTICS 的搜索距离相关。
    • Search Distance 搜索距离
      • 在 DBSCAN 中，如果某点在 Search Distance 内能找到最小要素数，它就是核心点。若某点本身不是核心点，但落在某个核心点搜索距离内，它是边界点 border point。若既不是核心点，也不在任何核心点搜索距离内，则是噪声点 noise point。

    空间约束多元聚类¶

    空间约束多变量聚类根据一组要素属性值，以及可选的簇大小限制，寻找空间上连续的要素簇。

    它与普通多变量聚类的区别在于：普通聚类只要求属性相似；空间约束多变量聚类还要求空间上相邻或接近。

    该算法采用连通性图（最小生成树）和称为SKATER的方法来查找数据中的自然聚类，以及证据积累来评估聚类成员关系。

    属于无监督学习

    添加了空间约束参数，可以确保生成的聚类在空间上相邻。可选的空间约束定义有仅邻接边、邻接边拐角、修剪型 Delaunay 三角测量，也可以自定义空间权重矩阵。
