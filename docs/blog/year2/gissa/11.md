# 地理空间大数据分析
## 地理空间大数据回顾
### 应用案例
- 全球光伏太阳能发电场清单
- 全球城市土地利用数据集 GULU
    - 提出了一种基于POI提示优化的城市土地利用制图框架：首先通过谷歌地球引擎（GEE）和开放街道地图（OSM）获取全球城市Sentinel-2影像、POI数据及标注的LU样本，进而构建POI提示城市土地利用制图网络（PPUL-Net）——该网络利用POI提示提升分类精度，并可在POI数据缺失区域生成可靠预测。基于此，全球首套2020年10米分辨率城市土地利用数据集（GULU）得以构建。
- 利用在线数据测量全球迁移流动

    - 研究利用约 42.9 万个样本点，结合 DEM 地形因子、MODIS 植被指数、气候数据等，通过回归模型估算全球树木密度

### ==地理空间大数据==的定义
!!! note
    简答题：概念与特征、和普通大数据的区别、arcgis工具举例

- 大数据是“大到传统数据库管理系统无法有效处理的数据集” (Dasgupta, 2013)。
- 地理空间/空间大数据（或，大地理数据）是指超出当前计算系统处理能力的空间数据集。
    - 对于大数据或空间大数据的最小规模没有标准阈值，尽管在2013年，大数据被认为是在1PB (1000 TB) 或以上 (Dasgupta, 2013)。

### 地理空间大数据的5V特征
- 速度快：数据增长快、处理速度要求快
- 价值低：数据价值密度相对较低
- 体量大：采集、存储和计算量庞大
- 真伪难辨：数据的准确性和可信度低
- 模态多：种类和来源多样化

大数据技术体系：数据、算法、算力、平台
### 地理空间大数据的数据源
- 用户生成内容 User-Generated Content
    - 地理空间大数据可以来自用户主动产生的信息，比如：社交媒体签到、评论、照片、地图标注、众包地理信息、OpenStreetMap 编辑、位置分享等。
    - 这些数据往往具有强烈的人类活动含义，可以用于研究城市功能、人口流动、旅游行为、灾害响应、空间感知等。
- 传感器数据 Sensor Data
    - 卫星遥感
    - 航空（有人或无人驾驶飞行器（UAV））测量
    - 雷达/激光雷达扫描
    - 传感器网络
    - 数码相机
    - RFID（射频识别）读数位置
    - 移动设备
    - 物联网
    - 模型，如 WRF, PLUS, CASE...
    - GNSS（全球导航卫星系统）- 包括北斗、GPS、伽利略...
    - 启用GNSS的设备
- 智能物体
    - 智能手机、智能汽车、智能手表、智能摄像头、智能建筑、智能交通设备、智能电网等都会不断产生带位置、时间和状态的数据。

### 地理空间大数据的挑战
- 数据库设计挑战
    - 存储大体量数据、支持高速读写和空间查询。
-  算法与方法挑战
    -  面对 5V 特征时，普通算法可能效率不足，需要并行化、分布式计算或近似算法。
- 网络传输挑战
    - 以大数据量或高速率传输数据
- 计算系统挑战
    - 普通桌面 PC 难以处理快速增长的大规模数据，需要服务器、超级计算机、云计算、分布式存储和大数据平台。
- 地理可视化挑战
    - 把大量多源数据快速可视化

## 地理空间大数据分析方法
### GeoAnalytics Server 工具
使用 Apache Spark 在桌面机器上提供并行处理框架。它可以完成汇总、聚合、回归、检测、聚类等任务，帮助用户从大数据中发现模式、趋势和异常。  

它和普通地理处理工具的区别是：普通工具多是单机串行或有限并行，而 GeoAnalytics Desktop Tools 面向更大规模数据，并且集成了 Spark 并行处理能力。  

- Summarize Data：汇总数据
    - 包括 汇总点、描述数据集、连接要素、汇总中心与离散度、汇总属性、范围内汇总、重新构建轨迹。
    - Aggregate Points汇总点工具把点数据聚合到多边形或网格中。输出结果通常包括每个区域内点的数量，也可以计算属性统计量。
    - Join Features连接要素：基于空间、时间或属性关系，或这些关系的组合，将属性从一个图层连接到另一个图层。
    - Recunstruct Tracks重新构建轨迹：根据带时间信息的点数据重建线轨迹或面轨迹。
    - Summarize Within 范围内汇总：它把一个多边形图层和另一个点、线或面图层叠加，然后统计每个多边形内部的点数量、线长度、面面积和属性统计量。
    - 这类工具用于统计、聚合、汇总和空间连接。
- Analyze Patterns：分析模式
    - 包括 计算密度、查找热点、查找点聚类、查找相似位置、基于森林的分类与回归、广义线性回归。
    - 这类工具用于发现空间分布规律、热点、聚类、相似性和回归关系。
- Use Proximity：邻近分析
    - 包括创建缓冲区、按邻近度分组。
    - 用于分析对象之间的距离、邻近关系和缓冲区影响。
- Find Locations：查找位置
    - 包括检测事件、查找相似位置、查找停留位置。
    - 适合事件检测、相似区域识别、轨迹停留点识别等。
- Manage Data：管理数据
    - 包括 计算字段、裁剪图层、叠加图层、融合边界。
    - 用于数据预处理和空间叠加管理。

### Analyze Tracks分析轨迹工具集
Analyze Tracks 工具集用于分析随时间采集的位置数据，也就是轨迹或移动数据。  


- 计算运动统计：计算运动统计量，如速度、加速度、方向变化等。
- 检测事件：创建一个检测满足给定条件的要素的图层。
    - 轨迹Track：由同一对象按时间排序形成的轨迹序列。例如一辆扫雪车每 10 分钟记录一次位置，车辆 ID 就是轨迹标识。
    - 时间Incident：满足感兴趣条件的事件。
    - 瞬间Instant：单个时间点，有开始时间但没有结束时间。
    - 时间间隔Interval：有开始和结束的时间段。
    - 感兴趣要素Feature of interest：被分析的要素
- 查找停留位置：使用给定的时间和距离阈值，查找移动对象停止或停留的位置。
    - 停留位置Dwell location：轨迹在给定时间和距离参数下保持静止或低移动的位置，输出可以是点、凸包或平均中心。
    - Track：时间启用的点序列。
    - 观测点Observation：轨迹中的一个点。
    - 测地线Geodesic：考虑地球曲率的测地线距离。
    - 平面Planar：平面欧氏距离。
    - Instant：单一时刻。
    - Interval：持续时间段。
- 查找相似位置： 根据属性找出与输入位置最相似或最不相似的候选位置。
- 重建轨迹：根据时间点重建线或面轨迹。
- 捕捉轨迹：将轨迹点匹配到道路或线网络
- 追踪邻近事件：追踪在空间（位置）和时间上彼此邻近的事件
