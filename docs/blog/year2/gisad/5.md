# 主流前端地图框架介绍与搭建
## WebGIS框架
Web Client → Web Server → GIS Server → GIS Database  


- Web client向Web server发送http请求
- Web server将GIS相关功能请求转发给GIS server
- GIS server从GIS database中读取所需数据，对请求进行处理
- GIS server将结果通过HTTP响应返回到client中显示

WebGIS客户端的类型  

- Web浏览器客户端
- 桌面应用程序客户端
- 移动客户端
    - 基于移动浏览器
    - 基于本地app
### 基于LBS地图的API
**Location-based Service（基于位置的服务，LBS）**通常需要在连接互联网的场景中使用，且使用地图服务商提供的数据和服务，基于LBS的地图API是一种通过 JavaScript（或其他语言）将地图嵌入到网页的 API。

代表：谷歌API、高德API、百度API  

??? note "example"
    - 应用
        - 外卖定位、打车导航、附近搜索、路径规划、商家位置展示、POI 查询
    - 特点：主要使用地图服务商提供的数据和服务。

### Charts
- 代表：D3.js,ECharts
- 本质上不是专门的 GIS 地图库，而是图表可视化库
- 常用于一些地图非页面表达主体的页面，只允许简单的交互，页面中同时还有其他类主题，有很强的图表可视化能力

## 前端地图框架介绍
### 历史
- 前H5时代
    - 前H5时代是**Flex，JS，Silverligh**t的“三驾马车”时代，这个时代JS还没有取得优势，产品都以**Flex**为首推。
    - 前H5时代WebGIS框架的代表有ArcGIS的**Flex API**（也有JS版）、开源的**OpenScale**（Openlayers2 是其JS版）
- H5时代
    - HTML5 标准发展；
    - JavaScript 统一前端；
    - Canvas 用于二维绘制；
    - WebGIS地图框架进入三维时代
    - MapboxGL发布
    - WebGL 用于 GPU 加速图形渲染；
    - WebGIS 框架开始重写或升级。
- WebGIS框架发展的特点
    - WebGIS前端地图框架，对应于Web技术和标准的发展，存在一致性和滞后性
    - WebGIS前端地图框架在产品设计上存在的缺陷和应对场景的不足是很难解决的

## Leaflet
Leaflet 是一个轻量级、开源的 JavaScript 地图库，主要用于构建移动端友好的交互式二维地图。  
具有便捷性、高性能和可用性，插件丰富，适合常规二维 WebGIS 开发  

核心类

| 类               | 作用                     |
| --------------- | ---------------------- |
| `Map`           | 地图对象，是整个 Leaflet 应用的核心 |
| `Marker`        | 标记点，比如地图上的图标           |
| `Popup`         | 弹窗，用来显示属性信息            |
| `Layer`         | 图层基类                   |
| `TileLayer`     | 瓦片图层，比如加载底图            |
| `TileLayer.WMS` | 加载 WMS 服务              |
| `ImageOverlay`  | 加载一张图片覆盖到地图上           |
| `Polyline`      | 线                      |
| `Polygon`       | 面                      |
| `Rectangle`     | 矩形                     |
| `Circle`        | 圆                      |
| `Canvas`        | 用 Canvas 渲染矢量图层        |
| `Controls`      | 控件，比如缩放、比例尺、图层切换       |


??? note "完整代码"
    ```html
    <!DOCTYPE html>
    <html lang="en">
    <head>
        <meta charset="UTF-8" />
        <meta name="viewport" content="width=device-width, initial-scale=1.0" />
        <title>Leaflet-Demo</title>

        <!-- 引入Leaflet的css和js包 -->
        <link
        rel="stylesheet"
        href="https://unpkg.com/leaflet@1.7.1/dist/leaflet.css"
        />
        <script src="https://unpkg.com/leaflet@1.7.1/dist/leaflet.js"></script>

        <!-- 设置leaflet地图容器大小与位置 -->
        <style>
        #map {
            position: absolute;
            left: 0;
            top: 0;
            right: 0;
            bottom: 0;
        }
        </style>
    </head>

    <body>
        <!-- 作为地图容器的div标签 -->
        <div id="map"></div>

        <!-- Leaflet脚本 -->
        <script>
        //创建地图对象；
        //设置地图中心点和缩放级别。
        var map = L.map("map").setView([76, 100], 4);

        // 矢量底图
        var tiandituVec = L.tileLayer(
            "https://t0.tianditu.gov.cn/vec_c/wmts?SERVICE=WMTS&REQUEST=GetTile&VERSION=1.0.0&LAYER=vec&STYLE=default&TILEMATRIXSET=c&FORMAT=tiles&TILECOL={x}&TILEROW={y}&TILEMATRIX={z}&tk=your_token"
        ).addTo(map);

        // 地图注记
        var tiandituCva = L.tileLayer(
            "https://t0.tianditu.gov.cn/cva_c/wmts?SERVICE=WMTS&REQUEST=GetTile&VERSION=1.0.0&LAYER=cva&STYLE=default&TILEMATRIXSET=c&FORMAT=tiles&TILECOL={x}&TILEROW={y}&TILEMATRIX={z}&tk=your_token"
        ).addTo(map);
        </script>
    </body>
    </html>
    ```


核心代码：

```js
// 1. 创建地图
var map = L.map("map").setView([76, 100], 4);

// 2. 加载底图
L.tileLayer("天地图矢量底图URL").addTo(map);

// 3. 加载注记
L.tileLayer("天地图注记URL").addTo(map);
```

- 优点
    - 主流投影坐标支持：支持几乎所有主流投影坐标系；
    - 矢量表达：矢量专题图，矢量空间分析，矢量瓦片，矢量可视化等;
    - 全样式表达：可结合主流Web可视化技术，如D3,Echarts,Mapv，实现几乎所有主要的地图可视化表达;
    - 功能全面，操作友好：插件丰富，社区生态完善，问题易解决；
    - 跨平台：兼容大部分浏览器，跨平台性强;
    - 移动设备：内部代码框架设计时就考虑了移动设备的支持。
- 缺点
    - 未使用WebGL渲染，可视化表达方面稍有欠缺；
    - 未使用硬件加速，没有发挥硬件的最大效果。

## OpenLayers
OpenLayers 也是一个开源 JavaScript 地图库，用来创建交互式地图。  

相比 Leaflet，OpenLayers 更偏专业 GIS。  

核心结构

| 组成          | 作用                  |
| ----------- | ------------------- |
| Map         | 地图对象                |
| Layer       | 图层                  |
| View        | 地图视图，控制中心点、缩放、投影、旋转 |
| Source      | 图层的数据来源             |
| Interaction | 交互，比如选择、绘制、修改       |
| Control     | 控件，比如缩放、比例尺         |

- 优点
    - 主流投影坐标系支持：支持几乎所有主流投影坐标系;
    - 脚本一体化：功能全面并且集成到官方脚本;
    - OGC协议：几乎是最遵循OGC协议的脚本;
    - 兼容性：兼容老的IE6789等疑难浏览器。
- 缺点
    - 许多功能有实现但是实际使用效果不理想;
    - 可视化表达欠缺;
    - 内存释放与优化欠缺。

## Mapbox GL JS
Mapbox GL JS 是一个使用 WebGL 渲染地图的 JavaScript 库。  
MapboxGL能够渲染大量的地图要素，拥有流畅的交互以及动画效果，可以显示立体地图并支持移动端，是用于**创建高性能交互式地图的先进解决方案**。  

核心类 

| 类型              | 作用              |
| --------------- | --------------- |
| `Map`           | 地图对象            |
| `Source`        | 数据源             |
| `Layer`         | 图层样式表达          |
| `CanvasSource`  | HTML Canvas 数据源 |
| `ImageSource`   | 图片数据源           |
| `VideoSource`   | 视频数据源           |
| `GeoJSONSource` | GeoJSON 数据源     |
| UI elements     | 地图控件            |

- 优点
    - 高效矢量瓦片：真正做到高效实用加载矢量瓦片;
    - 顶级可视化：实现顶级的可视化渲染效果;
    - 高清矢量：实现顶级的高清矢量图形绘制（SVG，Canvas）效果;
    - 尖端Web技术加持：拥有Baidu、Uber等顶级可视化巨头技术加持。
- 缺点
    - 空间分析能力相对欠缺;
    - 三维表达相对欠缺;

## Cesium
Cesium 是 Web 端**三维 GIS** 的代表框架。  
基于开放格式进行构建，以实现**强大的互操作性和大规模数据集扩展**。  

- 优点
    - 倾斜摄影：支持倾斜摄影、地形、海洋环境等三维场景展现;
    - BIM三维建模：支持BIM管网建模和3dx、gltf模型的展示;
    - 时态表达：支持时态、时间播放、时间动画、时空聚类等时空展现。
- 缺点
    - 没有类似unity的特殊光晕效果，虽然使用了WebGL但效果平平;
    - 自成体系的模型与几何绘制策略，需要重新学习;
    - 代码略重，并且主视图必须获取顶级div，影响工程代码结构。

### Viewer
```js
var viewer = new Cesium.Viewer('cesiumContainer');
```

Viewer 是 Cesium 应用的总入口，它负责创建三维地图窗口和一堆默认 UI 控件  

| 部分                | 作用                    |
| ----------------- | --------------------- |
| CesiumWidget      | 核心三维窗口                |
| Canvas            | 真正绘制三维场景的画布           |
| Scene             | 管理三维场景                |
| Camera            | 控制视角                  |
| Timeline          | 时间轴                   |
| Animation         | 动画控件                  |
| BaseLayerPicker   | 底图选择控件                |
| DataSourceDisplay | 把外部数据转成 Cesium 能显示的对象 |

### Entity
Entity 是 Cesium 中最常用的类型之一,Entity可以理解为地图/三维场景中的一个对象。  

- 主要负责
    - 基础地理环境设置；
    - 基础图层设置；
    - 场景数据；
    - 场景交互函数

`globe` 表示整个地球表皮。  


| 框架           | 核心特点               | 适合场景                       |
| ------------ | ------------------ | -------------------------- |
| Leaflet      | 轻量、简单、插件多          | 常规二维 WebGIS                |
| OpenLayers   | 专业、标准、OGC 支持强      | GeoServer、WMS/WFS、专业二维 GIS |
| Mapbox GL JS | WebGL、高性能、视觉效果好    | 高颜值地图可视化、矢量瓦片              |
| Cesium       | 三维地球、3D Tiles、时空动态 | 三维 GIS、倾斜摄影、BIM、地形         |

## 加载GeoServer
1,查看上节课发布的WMS图层，前端根据参数使用`WebMapServiceImageryProvider`类新建一个影像提供器。  
2.调用`viewer.imageryLayers.addImageryProvider()`方法，在viewer上添加图层  

```js
var viewer = new Cesium.Viewer("cesiumContainer");

const provider = new Cesium.WebMapServiceImageProvider({
    url:'http://localhost:8080/geoserver/工作区名/wms',
    layers:'工作区:图层名',
    parameters:{
        services:'WMS',
        transparent:true,
        format:'image/png'
    }
})

viewer.imageryLayers.addImageryProvider(provider);
```

| 代码                    | 含义                      |
| --------------------- | ----------------------- |
| `url`                 | GeoServer 的 WMS 服务地址    |
| `layers`              | 要加载的图层名称                |
| `parameters`          | WMS 请求参数                |
| `transparent: true`   | 设置图层背景透明                |
| `format: 'image/png'` | 返回 PNG 图片               |
| `addImageryProvider`  | 把这个 WMS 图层加到 Cesium 地球上 |
