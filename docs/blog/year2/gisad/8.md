# Cesium数据服务可视化与分析
## Cesium影像数据服务可视化
### 核心类
- **ImageryProvider**：影像数据源
    - `ImageryProvider`类是影像图层的数据源，提供了各种来源的数据。
    - `ImageryProvider`类本身是一个抽象类，不能被直接实例化。(不能直接`new Cesium.ImageryProvider()`，而是要使用它的子类)
    - Cesium提供了14种`ImageryProvider`类的子类，我们想使用哪种影像图层数据或服务就用对应的`ImageryProvider`子类去加载。
- **ImageryLayer**：真正的影像图层
    - `ImageryLayer`类用于表示影像图层，它的构造函数包含一个`imageryProvider`和一个配置对象。 ImageryLayer根据imageryProvider提供的数据源和亮度、透明度、对比度、饱和度配置创建一个影像图层。
    - alpha：透明度
    - brightness：亮度
    - contrast：对比度
    - hue：色相
    - saturation：饱和度
    - gamma：伽马值
```js
const layer = new Cesium.ImageryLayer(wmsProvider, {
    alpha: 0.8,
    brightness: 1.0,
    contrast: 1.0,
});
```
wmsProvider 提供数据，layer 控制怎么显示  


- **ImageryLayerCollection**：图层集合
    - `ImageryLayerCollection`类是`ImageryLayer`类对象的容器，它可以装载多个`ImageryLayer`类对象，并且这些对象是有序的。
    - Viewer类对象中的`imageryLayers`属性就是`ImageryLayerCollection`类的实例，它包含了当前Cesium应用程序所有的`ImageryLaye`r类对象，即所有影像图层，也就是说一个Viewer中可以加载多个影像图层。
    - `ImageryLayerCollection`类提供了一系列操作`ImageryLayer`的方法，可以实现对影像图层的增删改查。

viewer.imageryLayers = 当前地图上的所有影像图层  

- **ImageryLayerCollection**

| 方法                                    | 作用                                         |
| ------------------------------------- | ------------------------------------------ |
| `add(layer, index)`                   | 添加一个已经创建好的 ImageryLayer                    |
| `addImageryProvider(provider, index)` | 传入 provider，Cesium 自动帮你创建 ImageryLayer 并添加 |
| `remove(layer, destroy)`              | 删除图层                                       |
| `contains(layer)`                     | 判断集合中是否包含某图层                               |
| `get(index)`                          | 按索引获取图层                                    |
| `raise(layer)`                        | 图层上移一层                                     |
| `lower(layer)`                        | 图层下移一层                                     |

eg:然后把 Layer 加到 viewer.imageryLayers 里面   
```js
viewer.imageryLayers.addImageryProvider(provider);
```
### 影像数据加载
使用WebMapServiceProvider加载geosercer的wms数据  

```js
const wmsProvider = new Cesium.WebMapServiceImageryProvider({
    url: "http://localhost:8080/geoserver/webgis/wms",
    layers: "webgis:hangzhou_img",//表示加载哪个图层。
    parameters: {
        service: "WMS",//表示请求的是 WMS 服务
        format: "image/png",//返回 png 图片
        transparent: true,//背景透明
    },//请求参数
});
viewer.imageryLayers.addImageryProvider(wmsProvider);//把这个 WMS 数据源加到 Cesium 的影像图层集合里
```

使用WebMapTileServiceImageryProvider加载天地图矢量注记  
```js
var wmtsProvider = new Cesium.WebMapTileServiceImageryProvider({
    url: "http://t0.tianditu.gov.cn/cia_w/wmts?tk=你的天地图key",
    layer: "cia",//cia 通常表示中文注记图层。
    style: "default",
    titleMatrixSetID: "w",//w 一般表示 Web Mercator 瓦片矩阵集。
    format: "titles",//表示请求瓦片。
    maximumLevel:18,//最大缩放层级是 18
});
viewer.imageryLayers.addImageryProvider(wmtsProvider);
```

>WMTS 是瓦片地图服务,WMS 是按请求范围动态返回地图图片  
>WMS：你要哪一块，它临时给你画一张图，WMTS：提前切好很多小瓦片，你访问时直接拿。瓦片WMTS 通常加载更快，适合底图和注记。

使用OpenStreetMapImageryProvider加载OpenStreetMap在线地图  
```js
const osmProvider = new Cesium.OpenStreetMapImageryProvider({
    url: "https://tile.openstreetmap.org/",
})
viewer.imageryLayers.addImageryProvider(osmProvider);
```

使用UrlTemplateImageryProvider加载高德街道地图  
```js
var urlProvider = new Cesium.UrlTemplateImageryProvider({
    url: "http://webrd02.is.autonavi.com/appmaptile?lang=zh_cn&size=1&scale=1&style=8&x={x}&y={y}&z={z}",
});
viewer.imageryLayers.addImageryProvider(urlProvider);
```

使用IonImageryProvider加载Cesium官方提供的影像。需要先在个人中心中订阅相关的影像地图资产。  
```js
const ionLayer = Cesium.ImageryLayer.fromProviderAsync(
    Cesium.IonImageryProvider.fromAssetId(3)
);//ionLayer 是 ImageryLayer 类
viewer.imageryLayers.add(ionLayer);
```

固定格式
```js
// 1. 创建影像数据源 provider
const provider = new Cesium.某某ImageryProvider({
    url: "服务地址",
    // 其他参数
});

// 2. 添加到影像图层集合
viewer.imageryLayers.addImageryProvider(provider);
```

## Cesium地形数据可视化
Cesium提供了`TerrainProvider`基类，该Provider负责每一个Tile对应的地形数据的构建，定义了一套地形Provider需要实现的接口和规范，但本身并不会参与其中的操作。  

Cesium中的地形类由不同的`terrainProvider`直接控制的，把某一个实例化的`terrainProvider`赋值给`Viewer.terrainProvider`即可设置当前视图中的地形数据。  

### EllipsoidTerrainProvider：默认光滑椭球
默认采用的地形Provider，光滑椭球体，没有任何地形起伏效果，但不从服务器请求数据。一般用作一些太空类展示项目。  
```js
var ellipsoidProvider = new Cesium.EllipsoidTerrainProvider();
viewer.terrainProvider = ellipsoidProvider;
```
### ArcGISTiledElevationTerrainProvider
通过从ArcGIS ImageService的高程瓦片中提取的镶嵌高度图生成的地形几何。

### GoogleEarthEnterpriseTerrainProvider
通过Google Earth的高程地图方式生成的地形。

### CesiumTerrainProvider（标准地形，最常用）
高精度全球地形，支持光照和水面效果。支持量化网格地形切片，针对地形流进行了优化。   
```js
Cesium.CesiumTerrainProvider.fromIonAssetId(1)
    .then((terrainProvider) =>{
        viewer.terrainProvider = terrainProvider;//将terrainProvider赋值为viewer
    })
    .catch((e) => {
        console.log(e);//请求出错，输出报错
    });
```

开启水体和光照效果  
```js
Cesium.CesiumTerrainProvider.fromIonAssetId(1, {
    requestWaterMask: true, // 请求水体效果所需要的海岸线数据
    requestVertexNormals: true, // 请求地形照明数据
})
    .then((terrainProvider) => {
        viewer.terrainProvider = terrainProvider; // 将terrainProvider赋值给Viewer
    })
    .catch((e) => {
        console.log(e); // 如果请求出错，控制台输出错误信息
    });
```

??? note
    `Cesium.CesiumTerrainProvider.fromIonAssetId(1)`返回的不是普通的 terrainProvider，而是一个 Promise  
    一定要处理异步结果


## Cesium三维空间分析
Cesium 更侧重于三维可视化，对空间分析功能支持较弱。
### 空间量测
空间量测是在三维空间中量测距离、角度、面积、方向等内容。常见的实现方法主要是在屏幕中拾取对应点位置，然后将屏幕坐标转换为地理坐标，再根据地球椭球参数，进行几何解算，获取地理空间距离、空间面积等。  

```js
const point1 = new Cesium.Cartesian3.fromDegrees(120, 30, 0);
const point2 = new Cesium.Cartesian3.fromDegrees(121.3, 0, 0);
const distance = Cesium.Cartesian3.distance(point1, point2);
console.log("两点距离为：" + distance);
```

三角形面积估算：三角形面积 = 两条边向量叉乘结果的长度 × 1/2  
```js
function triangleArea(pnt0, pnt1, pnt2) {
    let vec0 = Cesium.Cartesian3.subtract(pnt0, pnt1, new Cesium.Cartesian3());//pnt0 pnt1坐标相减
    let vec1 = Cesium.Cartesian3.subtract(pnt2, pnt1, new Cesium.Cartesian3());
    let cross = Cesium.Cartesian3.cross(vec0, vec1, vec0);
    return Cesium.Cartesian3.magnitude(cross) * 0.5;
}
```

多边形面积估算：把多边形拆成多个三角形估算面积。  
以 points[0] 为公共点  
依次连接 points[i] 和 points[j]  
用叉乘累加面积  
```js
function polygonArea(points) {
    let area = new Cesium.Cartesian3(0, 0, 0);

    for (let i = 0; i < points.length; i++) {
        let j = (i + 1) % points.length;
        let vec0 = Cesium.Cartesian3.subtract(points[i], points[0], new Cesium.Cartesian3());
        let vec1 = Cesium.Cartesian3.subtract(points[j], points[0], new Cesium.Cartesian3());
        let cross = Cesium.Cartesian3.cross(vec0, vec1, vec0);
        area = Cesium.Cartesian3.add(area, cross, new Cesium.Cartesian3());
    }

    return Cesium.Cartesian3.magnitude(area) * 0.5;
}
```

### 淹没分析
淹没分析就是模拟水位上涨，看某个区域会被水淹到什么程度  

淹没分析主要利用polygon 的 `extrudedHeight` 属性   
`extrudedHeight` ：把 `polygon` 从地面向上拉伸到某个高度  

1.创建水体多边形  
```js
const waterPolygon = viewer.entities.add({
    name: "waterPolygon",
    polygon: {
        hierachy: Cesium.Cartesian3.fromDegreesArray([
            119.47, 28.17,
            119.47, 28.24,
            119.54, 28.24,
            119.54, 28.17,
        ]),//经纬度数组
        material: new Cesium.Color.fromBytes(64, 157, 253, 150),//fromBytes的参数是R,G,B,A,其中A是透明度，范围0-225
        extrudedHeight: 0,//初始水位
    },
});

viewer.zoomTo(waterPolygon);//视角飞到多边形
```
2.1使用setInterval改变水面高度
```js
let waterHeight = 0;
let targetHeight = 750;

setInterval(function(){
    waterHeight += 25;

    if(waterHeight > targetHeight){
        waterHeight = targetHeight;
        clearInterval(timer);//保存定时器并停止
    }

    waterPolygon.polygon.extrudedHeight = waterHeight;
},500);
```

- 每隔 500 毫秒执行一次函数
- 每次让 waterHeight 增加 25
- 如果超过目标高度，就固定为 targetHeight
- 然后把水体高度更新成 waterHeight

2.2使用 CallbackProperty 改变水面高度
```js
let waterHeight = 0;
let targetHeight = 750;

const altitudeProperty = new Cesium.CallbackProperty(() => {
    waterHeight += 0.5;
    if(waterHeight > targetHeight){
        waterHeight = targetHeight;
    }
    return waterHeight;
}, false);//false的意思是这个属性不是常量，会动态变化
waterPolygon.polygon.extrudedHeight = altitudeProperty;//把水体高度绑定为动态属性。
```
