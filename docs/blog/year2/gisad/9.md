# 09 Web应用前端开发框架
## 前端框架
### 前端框架概述
- 前端技术诞生之初的局限
    - 所有网页都基于HTML页面，无法控制局部显示隐藏，除非重新下载新页面
    - 计算服务只能在服务端实现
    - 页面静态
- 发展历史
    - HTML 静态页面
    - JavaScript 出现
    - iframe / AJAX 实现局部加载
    - jQuery 简化 DOM 操作
    - Angular / React / Vue 等前端框架出现

**前端框架**是一套已经封装好的网页开发工具，它帮你组织页面结构、管理数据变化、封装组件、处理交互，让开发更高效。  

- 作用
    - 提升美观度
    - 减少工作量
    - 高效
    - 兼容性好

### Angular
Angular 是 Google 维护的前端框架，它的特点是大而全  

- 优势：
    - 背靠Google，有稳定的维护团队
    - 相对react和vue而言更加完整。自带了几乎所有页面应具有的功能，路由、表单、ajax、模版、双向数据绑定等等
    - 每个组件都是一个文件夹，html、js、和css文件分离，代码清晰
- 劣势：
    - 向数据绑定和变更检测机制在超大型应用中可能引发性能瓶颈。
    - 学习成本较高，api文档内容过多，阅读困难

### Vue
Vue 是一个渐进式框架，核心库只关注视图层。  

- 优势：
    - 简单。官方文档清晰，比Angular简单易学。
    - 快速。异步批处理方式更新DOM。
    - 组合。用解耦的、可复用的组件组合应用程序。
    - 紧凑。体积小，且无依赖。Vue 3.0核心库体积约20KB。
    - 对模块友好。可以通过NPM、Bower或Duo安装，不强迫所有的代码都遵循Angular的各种规定，使用场景更加灵活。
- 劣势：
    - 兼容性一般。要求浏览器支持es5，所以不支持ie8及以下浏览器

### React
React 是 Facebook 推出的一个用于构建用户界面的 JavaScript 库  
React 主要负责 UI 视图层，擅长解决数据变化后，页面应该怎么高效更新  

- 优势：
    - 速度快。在UI渲染过程中，React基于虚拟DOM，通过微操作来实现对实际DOM的局部更新。
    - 跨浏览器兼容。虚拟DOM帮助我们解决了跨浏览器问题，它为我们提供了标准化的API，甚至在IE8中都是没问题的。
    - 模块化。为程序编写独立的模块化UI组件，当某个或某些组件出现问题时，可以方便地进行隔离。
    - 单向数据流，数据变化可预计。
- 劣势：
    - React本身不够完整，写大型应用通常需要与其他库结合。

**React和普通JavaScript的区别**  

普通 JavaScript 中，我们直接操作真实 DOM：
```js
const box = document.getElementById('box');
box.innerHTML = 'Hello';
```

React 中，我们通常不直接写 DOM 操作，而是写组件：
```js
function App() {
    return (
        <div>Hello</div>
    );
}
```

**虚拟DOM**  

React 会先将代码转换成一个 JavaScript 对象，然后这个 JavaScript 对象再转换成真实 DOM，这个 JavaScript 对象就是虚拟 DOM。  
虚拟 DOM 不是真的页面元素，而是一个普通 JS 对象。  

??? example
    例如JSX：
    ```js
    <h1>Hello React</h1>
    ```
    可以理解成被React转成类似这样的对象：  
    ```js
    {
        type: 'h1',
        props: {
            children: 'Hello React'
        }
    }   
    ```
    这个对象不是真正显示在页面上的东西，只是 React 用来描述页面结构的数据  


#### 项目基础结构
```
index.html  
main.jsx  
App.jsx  
```

**index.html**  
index.html 是页面的最外层 HTML。可以写成简单形式  
```html
<!DOCTYPE html>
<html>
<head>
    <meta charset="UTF-8">
    <title>React App</title>
</head>
<body>
    <div id="root"></div>
    <script type="module" src="/src/main.jsx"></script>
</body>
</html>
```
重点是`<div id="root"></div>`，root是React挂在的位置，可以理解为React整个项目最后都会被塞进这个div里面  

**main.jsx**  
```js
import React from 'react';
import ReactDOM from 'react-dom/client';
import App from './App.jsx';

ReactDOM.createRoot(document.getElementById('root')).render(
    <App />
);
```

`ReactDOM.createRoot(document.getElementById('root'))`:找到HTML里的root节点，把它作为React应用的根节点  
`.render(<App/>)`：把App组件渲染到root里  

**App.jsx**  
App.jsx是项目主体结构
```js
function App() {
    return (
        <div>
            <h1>我的 WebGIS 项目</h1>
        </div>
    );
}

export default App;
```

#### JSX语法：
- 全称为JavaScript XML。JSX 是一个 JavaScript 的语法扩展，它把xml当做变量的值赋给JS变量，这是对JavaScript语法的延伸，因此它具有 JavaScript 的全部功能。
- 使用JSX语法可以简化创建虚拟DOM
    - 例： l`et ele = <h1>Hello JSX!</h1>`
- 实现一个React组件时，可以使用JSX格式来实现。

1.return 后面通常只能返回一个根元素  

2.JSX 里写 JavaScript 表达式要用 {}  
```js
function App() {
    const name = 'Cesium';

    return (
        <h1>Hello {name}</h1>
    );
}
```
页面显示`Hello Cesium`  

3.class 要写成 className  
HTML写法：`<div class = "box"></div>`  
React JSX写法：`<div className = "box"></div>`  

4.事件写法是驼峰命名  
HTML写法：`<button onclick = "test()">点击</button>`  
React写法：`<button onClick = {test}>点击</button> `

#### 组件
组件是用来实现局部功能效果的代码和资源的集合，包括 html、css、js、image 等，就是把页面中的一块功能单独封装起来。  

用组件：  
```js
function LayerButton(props) {
    return (
        <button>{props.name}</button>
    );
}
```

使用：
```js
<LayerButton name="影像图层" />
<LayerButton name="地形图层" />
<LayerButton name="建筑图层" />
```

组件包括函数组件和类组件，现在推荐使用函数组件  
函数组件：  
```js
function Hello() {
    return (
        <h1>Hello React</h1>
    );
}

export default Hello;
```
类组件：  
```js
import React from 'react';

class Hello extends React.Component {
    render() {
        return (
            <h1>Hello React</h1>
        );
    }
}

export default Hello;
```

#### state
React 把组件看成一个状态机。state 就是组件内部会变化的数据。  

函数组件中使用`usrState`  
```js
import { useState } from 'react';

function Counter() {
    const [count, setCount] = useState(0);//初始值是0

    function add() {
        setCount(count + 1);//修改count的函数
    }

    return (
        <div>
            <p>当前次数：{count}</p>
            <button onClick={add}>点击增加</button>
        </div>
    );
}

export default Counter;
```

#### props
props 用于从组件外向组件内传递变化的数据。  
父组件给子组件传参数  
state 和 props 主要的区别在于 props 是不可变的，而 state 可以根据与用户的交互发生改变  

#### 生命周期
组件从创建到死亡它会经历一些特定的阶段，React组件中包含一系列钩子函数(生命周期回调函数)，会在特定的时刻调用。我们在定义组件时，会在特定的生命周期回调函数中，做特定的工作

#### 钩子函数Hook
Hook 是 React 16.8 的新增特性，主要为了支持函数式组件的写法。它可以让你在不编写 class 的情况下使用 state 以及其他的 React 特性。

## UI框架
- UI（User Interface），即用户界面，主要包含视觉（比如图像、文字、动画等可视化内容）以及交互（比如按钮点击、列表滑动、图片缩放等用户操作）。
- UI框架，则是为开发UI而提供的基础设施，比如视图布局，UI组件，事件响应机制等。


- Bootstrap：Twitter推出的一个用于前端开发的开源工具包，是全球最受欢迎的前端组件库，用于开发响应式布局、移动设备优先的 WEB 项目。
- Material-UI：基于React 的UI框架，用于更快速、更简便的 web 开发。
- Element Plus：一个Vue 3 UI框架，是基于Vue 3.0 的组件库，提供了配套设计资源，帮助网站快速成型。
- Ant Design：基于 Ant Design 设计体系的 React UI 组件库，主要用于研发企业级中后台产品。
