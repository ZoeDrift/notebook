# 后端开发框架

## 概述
经典架构：分层耦合：  

* 表现层：接受HTTP请求，处理输入校验与响应格式化
* 业务逻辑层：封装核心业务规则
* 数据逻辑层：隔离数据库操作
* 数据库：存储数据

* Gin

## FastAPI
### 最简样例
基于FastAPI框架搭建的一个简单的后端接口大致如下：
```python
from fastapi import FastAPI 
import uvicorn # 导入uvicorn，用于启动应用

app = FastAPI(title="WebGIS") # 创建FastAPI应用实例
@app.get("/") # 注册路由
async def root(): # 路由处理函数
    return {"message": "欢迎使用WebGIS"}

if __name__ == "__main__":
    uvicorn.run(app, host="127.0.0.1", port=8000) # 启动应用

```

Uvicorn 是一个 ASGI 服务器。FastAPI负责写接口，Uvicorn负责运行接口。  

#### 路由
路由：指的是第一个“/”开始的URL的最后一部分。路由是URL与函数之间的映射关系。通过定义路径操作装饰器，说明其下面的函数对应的路径参数和操作    
例如`https://example.com/items/foo`  中`items/foo`  

```python
@app.get("/")
def home():
    return {"message": "首页"}
@app.get("/items")
def get_items():
    return {"message": "商品列表"}
@app.get("/map")
def get_map():
    return {"message": "地图数据"}
```

|用户访问|	执行哪个函数|
|-----|------|
|`/`|home()|
|`/items`|	get_items()|
|`/map`|	get_map()|

#### 处理函数
```python
async def root():
    return {"message": "欢迎使用WebGIS"}
```
返回的是 Python 字典，但 FastAPI 会自动把它变成 JSON：
```json
{
  "message": "欢迎使用WebGIS"
}
```

#### 启动服务
```python
uvicorn.run(app, host="127.0.0.1", port=8000)
```

FastAPI 本身负责“定义接口”，Uvicorn 负责“把接口跑起来”


### 路径参数
路径参数：写在 URL 路径里的变量

```python
@app.get("/items/{item_id}")
async def root(item_id):
    return {"message": item_id}
```
假设访问：`https://example.com/items/foo`，那么 {item_id} 就会接收到 "foo"。所以返回：  

```json
{
  "message": "foo"
}
```

这里的 {item_id} 就是路径参数。它适合用来表示“我要查哪一个资源”。  



比如 WebGIS 里： 
```python
@app.get("/layers/{layer_id}")
def get_layer(layer_id):
    return {"layer_id": layer_id}
```
访问：`/layers/roads`意思就是：我要查`roads`这个图层。  
访问：`/layers/buildings`意思就是：我要查`buildings`这个图层。

### 查询参数
查询参数：写在`?`后面的参数

```python
items_db = {
    "foo": [f"foo{i}" for i in range(100)],
    "bar": [f"bar{i}" for i in range(200)]
}
@app.get("/items/{item_id}")
def root(item_id, skip: int, limit: int = 10):
    return {"message": items_db[item_id][skip : skip + limit]}
```
对应 URL 是：`https://example.com/items/foo?skip=0&limit=10`

|部分|含义	|
|---|---|
|`/items/foo`|	路径|
|`foo`|	路径参数 item_id|
|`?skip=0&limit=10`|查询参数|
|skip=0|从第几个开始取|
|limit=10|取多少个|

所以这个接口的意思是：从 `items_db["foo"]` 里面，从第 0 个开始，取 10 个。也就是`items_db["foo"][0 : 0 + 10]`其中有默认值的参数为可选，其余为必选

查询参数适合做筛选、分页、排序，不适合表示资源本身。

比如
```python
@app.get("/poi")
def get_poi(category: str, limit: int = 10):
    return {"category": category, "limit": limit}
```
访问：`/poi?category=restaurant&limit=20`意思是：查询餐馆类 POI，最多返回 20 条。  

* 路径参数表示“你要哪个东西”。比如：`/items/foo`意思是我要 foo 这个资源。
* 查询参数表示“怎么查、查多少、怎么筛”。比如：`/items/foo?skip=0&limit=10`意思是我要 foo 这个资源，但是只取第 0 到第 10 条。

### 请求体
请求体 Body：POST 提交复杂数据

GET 请求一般把参数写在 URL 里，如`/poi?category=food&limit=10`

但是如果数据很多，就不适合放 URL 里了。比如注册一个读者：
```json
{
  "id": "R0001",
  "name": "Reader",
  "contact": "xxx-xxxx-xxxx",
  "max_books": 3
}
```
这种数据就应该放在请求体 Body 里。

FastAPI 用 Pydantic 模型定义请求体结构：
```python
from pydantic import BaseModel
class Reader(BaseModel):
    id: str
    name: str
    contact: str
    max_books: int = 3
```
然后接口这样写：
```python
@app.post("/readers/")
def create_reader(reader: Reader):
    return reader
```
这里的 reader: Reader 意思是前端必须传一个符合 Reader 结构的 JSON。其中有默认值的参数为可选，其余为必选


Pydantic 的作用 ￼

* 检查数据格式。
* 自动转换数据。
    * 如果前端传："max_books": "3",FastAPI 可能会帮你转成整数 3。
* 自动生成 API 文档。
    * 定义了 Reader 模型之后，FastAPI 会在 /docs 里自动展示这个接口需要哪些字段。



### 数据验证：Query、Path、Field

|函数|管什么|
|---|---|
|Query|查询参数|
|Path|路径参数|
|Field|请求体模型里的字段|

eg：


查询参数验证：
```python
from fastapi import Query
@app.get("/items")
def get_items(keyword: str = Query(default=None, min_length=3, max_length=50)):
    return {"keyword": keyword}
```
keyword 可以不传；如果传了，长度必须在 3 到 50 之间。


路径参数验证：
```python
from fastapi import Path
@app.get("/items/{item_id}")
def get_item(item_id: int = Path(gt=0)):
    return {"item_id": item_id}
```
item_id 必须是大于 0 的整数。

请求体字段验证：
```python
from pydantic import BaseModel, Field
class Reader(BaseModel):
    name: str = Field(min_length=2, max_length=20)
    max_books: int = Field(default=3, ge=1, le=10)
```
name 长度必须是 2 到 20。  
max_books 默认是 3，最小 1，最大 10。  

### 依赖注入 Depends
依赖注入是声明代码运行所需的依赖，由 FastAPI 负责执行逻辑并提供依赖。简单说就是把一些重复逻辑单独写成函数，然后让 FastAPI 自动帮你调用。  

```python
def Item_select(skip: int = 0, limit: int = 10)://这个函数负责处理分页参数。
    return {"skip": skip, "limit": limit * 2}

from fastapi import Depends
@app.get("/items/{item_id}")
def root(item_id, item_select = Depends(Item_select)):
    skip = item_select["skip"]
    limit = item_select["limit"]
    return {"message": items_db[item_id][skip : skip + limit]}
```
访问接口时，FastAPI 会先执行 Item_select()，拿到它的返回值，再传给 item_select。  

所以如果访问：`/items/foo?skip=0&limit=10`Item_select() 会得到：`skip = 0` `limit = 10`，然后返回：
```json
{
    "skip": 0,
    "limit": 20
}
```
最终会取 20 条数据。

依赖注入最常见的用途：

|用途|例子|
|----|----|
|数据库连接|每个接口都需要数据库|
|用户认证|判断用户有没有登录|
|权限检查|判断用户能不能访问某图层|
|分页参数|统一处理 skip 和 limit|
|公共配置|读取统一配置项|

```python
def get_db():
    db = connect_database()
    return db
@app.get("/layers")
def get_layers(db = Depends(get_db)):
    return db.query("select * from layers")
#不用在每个接口里都重复写连接数据库的代码。
```

## RESTful 是什么？
URL设计原则
HTTP协议  
`协议://IP地址或域名[:端口]/路径参数[?查询参数][#锚]`


RESTful 是一种面向资源设计的 API 风格。 ￼
在 RESTful 里，URL 尽量表示“东西”，而不是“动作”。  
例如：  

|操作|RESTful 写法|
|获取所有图层| `GET /layers`|
|获取某个图层|`GET /layers/{id}`|
|新建图层|`POST /layers`|
|修改图层|`PUT /layers/{id}`|
|删除图层|`DELETE /layers/{id}`|

动作交给 HTTP 方法，资源交给 URL。

```python
@app.get("/layers")
def get_layers():
    pass
@app.post("/layers")
def create_layer():
    pass
@app.get("/layers/{layer_id}")
def get_layer(layer_id: int):
    pass
@app.put("/layers/{layer_id}")
def update_layer(layer_id: int):
    pass
@app.delete("/layers/{layer_id}")
def delete_layer(layer_id: int):
    pass
# 这里 URL 都是 /layers，但是方法不同，含义不同。
```

* HTTP 方法
  * GET	查询
  * POST 新增
  * PUT	更新
  * DELETE 删除


## API文档

### API 文档：/docs 和 /redoc

* `/docs`
    * `http://127.0.0.1:8000/docs`
    * 这是 Swagger UI，可以直接在网页里测试接口。
* `/redoc`
    * `http://127.0.0.1:8000/redoc`
    * 这是另一种文档展示方式，更适合阅读。

FastAPI 会根据代码自动生成文档，比如：
```python
@app.get("/items/{item_id}")
def root(item_id: str, skip: int, limit: int = 10):
    return {"message": "ok"}
```
文档里会自动显示：

|参数|来源|类型|是否必填|
|----|----|----|----|
|item_id|路径参数|string|必填|
|skip|查询参数|integer|必填|
|limit|查询参|integer|可选，默认 10|

把这一部分串起来，一个 FastAPI 接口大概可以这么理解：
```python
@app.get("/items/{item_id}")
def root(item_id: str, skip: int = 0, limit: int = 10):
    return {"message": items_db[item_id][skip : skip + limit]}
```
它包含了：

* @app.get(...)	
    * 注册 GET 接口
* "/items/{item_id}"
    * URL 路径
* {item_id}	
    * 路径参数
* item_id: str	
    * 接收路径参数
* skip: int = 0	
    * 查询参数，默认 0
* limit: int = 10	
    * 查询参数，默认 10
* return {...}	
    * 返回 JSON 响应

访问：`/items/foo?skip=10&limit=5`FastAPI 会执行：`items_db["foo"][10 : 15]`
